embedding versus inheritance

package main

import "fmt"

type Inner struct{}

type Outer struct{ Inner }

func (i Inner) F() string {
	return i.G()
}

func (i Inner) G() string { return "inner" }

func (o Outer) G() string { return "outer" }

func main() {
	o := Outer{Inner{}}
	fmt.Println(o.F())
}

inner

class Parent:
    def f(self):
        return self.g()

    def g(self):
        return "parent"


class Child(Parent):
    def g(self):
        return "child"


c = Child()
print(c.f())

child

Inner and Outer's relationship is like and unlike Parent and Child's.

Inner is an embedded field in Outer. Fields or methods declared on an embedded field are promoted to the containing struct. So we can call o.F() ok.

Parent is an ancestor of Child. Attributes defined on an ancestor are available to its descendants. So we can call c.f() ok.

In go, promoted methods don't know they're embedded. So i.G() returns "inner" even when called via o.F().

In python, methods know their receiver: it's stored in self. So self.g() returns "child" when the receiver is c.
