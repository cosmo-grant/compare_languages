slicing out of bounds

const arr = [0, 1, 2, 3, 4]
console.log(arr.slice(2, 50))

[ 2, 3, 4 ]

package main

import "fmt"

func main() {
  s := []int{0, 1, 2, 3, 4}
  fmt.Println(s[2:50])
}

panic: runtime error: slice bounds out of range [:50] with capacity 5

lst = [0, 1, 2, 3, 4]
print(lst[2:50])

[2, 3, 4]
python returns any items with indices in the given range. This is an example of what John Ousterhout calls "defining errors out of existence". Go is stricter.
