Special syntax to define a generator in javascript.
The generator is done after the last yield but at a return.
You can keep fetching from a done generator.

Just a function with a yield in python.
A return in a generator raises a StopIteration.
The return value is smuggled out via StopIteration.value.
The last `print` never runs, but if it did (say the second `print` is in a `try/except`) then it would just be `StopIteration`, no value!
