A language has type coercion if it implicitly converts a value of one type to another type in certain situations. A language has operator overloading if an operator's implementation can depend on its operands' types.

Javascript has type coercion. For example, if one of +'s operands is a string, Javascript implicitly converts the other to a string too. String addition is concatenation. Javascript doesn't have operator overloading.

Python has operator overloading. For example, to evaluate x + y, Python first tries x.__add__(y), and falls back on y.__radd__(x). (There's a good diagram covering all cases in Fluent Python, 2nd Edition, p. 573.) Python doesn't have type coercion.

Type coercion and operator overloading here enable the same behaviour: adding values of different types. But they're quite different. Do languages tend to have at most one of the two? What are the tradeoffs?
