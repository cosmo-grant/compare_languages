Surely Python is more sensible here. In Node, if you want an array containing your string, clearer to write [s]. If the separator is passed in, clearer to write something like sep ? s.split(sep) : [s].

MDN gives this example: 

const strCopy = str.split();
console.log(strCopy);
// Expected output: Array ["The quick brown fox jumps over the lazy dog."]

An odd example, because strCopy is not a string, and copying is moot for immutables. It doesn't explain Node's default behaviour.
