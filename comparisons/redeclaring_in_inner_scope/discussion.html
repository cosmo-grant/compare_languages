Javascript, Go and Python all resolve a variable name by checking enclosing scopes in order. So they all print 1 ok. But they differ in how variables are hoisted.

In Javascript, variables declared with let are hoisted: i.e. if you declare a variable then all occurrences of the name in that scope, including any occurrences before the declaration, refer to that variable. But the variable is not initialized until you assign to it. So reading a variable before its declaration throws an error.

There's no hoisting in Go. If you declare a variable then all subsequent occurrences of the name in that scope refer to that variable. But previous occurrences are unaffected. So the first fmt.Println(y) finds the package block's y, and the second finds the function block's.

In Python, you don't declare variables: you just read or assign to them. If you assign to a variable then all occurrences of the name in that scope, including any occurrences before the assignment, refer to that variable. So reading a variable before its assignment throws an error.

A qualification. In Python, the nonlocal and global keywords do let you reassign variables in outer scopes. If we added global y at the top of main(), then no error, and y in the global scope would end up assigned to 3. In Javascript, you can achieve the same effect just by not declaring the variable in the current scope.

The key point: in Javascript and Python, either all or no occurrences of a variable name refer to a local variable; but in Go, initial occurrences can refer to a non-local variable and later occurrences to a local variable.
