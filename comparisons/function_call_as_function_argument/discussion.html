When is foo() called?

In Node, it's called when bar() is called without passing a value for x. Defaults are evaluated per call, when necessary.

In Python, it's called once, at definition time. (Its return value, None, is stored in foo.__defaults__.) Defaults are evaluated per function, no matter what.

Mutable defaults are a bad idea in Python. (Calls share the default, so see each other's mutations.) Setting defaults via function calls is a bad idea too. (Calls share the default, so the default-setting function only runs once.) Two aspects of the same language feature.

In a code review once I saw something like

def create_customer(id=uuid4()): ...

dooming us never to land a second sale.
